# 个人运营系统需求与迭代计划

## 产品定位

做一个“个人运营中枢”：每天一打开先看到自己的状态、任务和提醒，再按人际、运动、学习、工作、理财、出行六个模块沉淀信息。数据优先归自己所有，后续同步到 OneDrive。

## 第一阶段：能用的本地原型

- 首页看板：运动量、睡眠、待办、支出。
- 人际关系：新增人物、关系分类、简介、事件、喜好、文件名记录、CSV 导入、关系图拖拽。
- 运动：身高体重、心率、步数、热量、睡眠手动录入，体重趋势。
- 学习：英语模块先留需求笔记。
- 工作：重点任务、进度、时间节点、按时间段生成总结 Prompt。
- 理财：流水手动录入、CSV 导入、截图待识别占位、审核补全。
- 出行：记录和规划两类，支持名称、分类、定位、时间、照片/文件名。

## 第二阶段：正式移动 App

- 使用 Capacitor 封装 Android APK。
- 本地数据库从 localStorage 升级为 SQLite。
- 文件附件保存到本地 App 目录。
- 支持数据备份、恢复、导出。

## 第三阶段：外部数据接入

- OneDrive：用 Microsoft Graph 同步 JSON 和附件。
- 高驰：先确认官方 API 或数据导出能力；没有 API 时做“导出文件导入”。
- 微信/支付宝：优先支持官方账单 CSV，截图 OCR 后置。
- 地图定位：Android 定位权限 + 地图 SDK 或系统地图跳转。

## 需要继续确认的问题

1. 首页看板最重要的 4 个指标固定为：运动、睡眠、任务、支出，还是你想换成“工作风险/待办/健康/支出”？
2. 人际关系里的“事件”更像日记，还是更像 CRM 跟进记录？是否需要提醒日期？
3. 工作模块要不要区分“工作任务、材料写作、指标问题、会议事项”四类？
4. 理财模块你更看重预算控制、消费分类，还是资产净值变化？
5. OneDrive 同步是“自动同步”，还是“手动点一下备份/恢复”更符合你的习惯？
6. APK 第一版是否只服务你自己？如果是，登录体系可以先不做。
